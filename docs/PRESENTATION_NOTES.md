## Key Architectural Decisions

### Why It Scales

1. **Dual Registry Pattern** - Static skills (curated) + Dynamic skills (user-created)
2. **Streaming-First** - All AI providers configured for real-time output
3. **Local-First Storage** - IndexedDB with cloud sync optional
4. **Component Composition** - Reusable RunnerLayout, FormBuilder, OutputPanel

### Why It's Fast

1. **Vite** - Sub-second HMR during development
2. **Route-based code splitting** - Only load what's needed
3. **No heavy state libraries** - React Context + custom hooks
4. **Streaming responses** - Users see output immediately

### Why It's Maintainable

1. **TypeScript everywhere** - Full type safety
2. **Clear file organization** - pages/, components/, lib/, hooks/
3. **Comprehensive tests** - 91 test cases for critical paths
4. **Documentation** - Architecture.md, DESIGN_DOCUMENT.md, TESTING.md
